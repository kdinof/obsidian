## üîç Analysis Criteria  
Use this list as a QA walk-through: start at #1 and tick off each sub-bullet.

1. Visual hierarchy ‚Äî make the primary action unmistakable
   ‚Ä¢ Single focal call-to-action: one button should dominate.  
   ‚Ä¢ Limit distractions: keep only the controls that serve the main task.  
   ‚Ä¢ Target sizing & distance: larger, closer tap targets (Fitts's Law).  
   ‚Ä¢ Chunk options: present no more than ~7 choices at once or group them logically.  
   ‚Ä¢ Reduce cognitive load: remove elements that don't advance the goal.  

2. Composition & grid ‚Äî rhythm that scales
   ‚Ä¢ Responsive layout: switch between 1‚Äì3 content panes based on viewport width.  
   ‚Ä¢ Consistent spacing tokens: follow a 4/8-point scale for padding and gaps.  
   ‚Ä¢ Safe areas: keep content clear of notches, status bars, and home indicators.  

3. Typography ‚Äî legible at any size
   ‚Ä¢ Use predefined text styles to create instant hierarchy.  
   ‚Ä¢ Text must scale to 200 % without clipping or overlap.  
   ‚Ä¢ Text-spacing tolerance: line-height ‚â• 1.5√ó, paragraph-spacing ‚â• 2√ó, no layout breakage.  

4. Color & contrast ‚Äî readable, brand-aligned
   ‚Ä¢ Minimum contrast ratio 4.5 : 1 for body text; 3 : 1 for large/bold text.  
   ‚Ä¢ Aim for 7 : 1 contrast on critical UI pairs for extra clarity.  
   ‚Ä¢ Define semantic roles (primary, on-primary, surface, error) to support theming and dark mode.  
   ‚Ä¢ Never rely on color alone to convey meaning; back it up with text or icons.  

5. Components & patterns ‚Äî consistency over novelty
   ‚Ä¢ Prefer native or well-known patterns; they come with built-in accessibility.  
   ‚Ä¢ Document every state of custom components (default, hover, focus, pressed, disabled).  
   ‚Ä¢ Ensure cross-screen consistency: same component = same name, icon, and behavior.  
   ‚Ä¢ Verify each component state meets minimum contrast ratios.  

6. Ux flow ‚Äî clear steps, graceful failure
   ‚Ä¢ Error messages: display inline, next to the field, explaining how to fix the issue.  
   ‚Ä¢ Inline guidance beats pop-ups: show contextual help near the control.  
   ‚Ä¢ Empty & loading states: always communicate purpose and next action ("Add item", "Retry").  
   ‚Ä¢ Progressive disclosure: break complex tasks into smaller, sequential steps.  
   ‚Ä¢ Micro-copy: concise, action-oriented, free of jargon, aligned with the screen's key KPI.
